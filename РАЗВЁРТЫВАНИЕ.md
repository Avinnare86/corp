# Публикация на GitHub, обновление и развёртывание

Проект — **портативное** PHP-приложение: PHP лежит внутри (`tools\php`), база — SQLite
(`storage\database.sqlite`). Установка ничего не требует — на новом ПК достаточно `git clone`
и запуска `start.bat`.

Репозиторий уже инициализирован (`git init`, ветка `main`, первый коммит сделан).
Осталось один раз создать репозиторий на GitHub и залить.

---

## 1. Первая публикация на GitHub (один раз)

### Что НЕ попадёт в GitHub (так задумано — см. `.gitignore`)
- `storage\secret.key` — ключ шифрования настроек (секрет);
- `*.bas` — файлы со встроенным ключом OpenRouter (секрет);
- `storage\database.sqlite` — ваши рабочие данные (создаётся заново миграцией);
- `storage\uploads`, `storage\tmp` — загрузки и временные файлы;
- `*.rar`, `*.zip`, `*.pdf` и документы-образцы в корне (содержат персональные данные).

> ⚠️ Рекомендуется **приватный (Private)** репозиторий — это внутренний инструмент организации.

### Вариант А — через сайт GitHub (проще всего)
1. Зайдите на https://github.com/new и создайте **пустой** репозиторий
   (например имя `uchet`), **без** галочек «Add README / .gitignore / license».
2. Выберите видимость **Private**.
3. В консоли (PowerShell) в папке проекта:
   ```powershell
   cd E:\uchet
   git remote add origin https://github.com/ВАШ_ЛОГИН/uchet.git
   git push -u origin main
   ```
4. При запросе пароля вставьте **Personal Access Token** (не обычный пароль):
   GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) →
   Generate new token → права **repo** → скопировать и вставить вместо пароля.

### Вариант Б — через GitHub CLI (если установите `gh`)
```powershell
winget install GitHub.cli      # установка (один раз), затем перезапустить консоль
gh auth login                  # вход в GitHub (выбрать HTTPS, открыть браузер)
cd E:\uchet
gh repo create uchet --private --source=. --remote=origin --push
```

### Вариант В — GitHub Desktop (без консоли)
Установить GitHub Desktop → File → Add local repository → указать `E:\uchet` →
Publish repository (галочка «Keep this code private») → Publish.

## 2. Заливка изменений (каждый раз после правок)

```powershell
cd E:\uchet
git add -A
git commit -m "Коротко: что изменил"
git push
```

В GitHub Desktop: написать summary → Commit to main → Push origin.

---

## 3. Развёртывание на другом компьютере

```powershell
git clone https://github.com/ВАШ_ЛОГИН/uchet.git
cd uchet
start.bat
```
`start.bat` при первом запуске сам создаст базу (миграция + демо-данные) и откроет
http://localhost:8000. Портативный PHP уже в репозитории — ставить ничего не нужно.

**Демо-входы** (после первого запуска): `admin / admin123`, менеджер виз `vizman / vizman123`,
специалист `petrov / petrov123`, контролёр `control / control123`.

> Чтобы перенести **свои** данные (а не демо) — скопируйте на новый ПК файлы
> `storage\database.sqlite` **и** `storage\secret.key` вручную (они не в git). Без `secret.key`
> зашифрованный ключ OpenRouter в БД не прочитается — его можно просто ввести заново в `/admin/settings`.

---

## 4. Обновление уже установленной копии

```powershell
cd путь\к\uchet
git pull
tools\php\php.exe database\migrate.php
```
`migrate.php` безопасно добавит новые таблицы/колонки (идемпотентно, данные не теряются).
Затем перезапустить `start.bat`.

---

## 5. Резервная копия данных

Все рабочие данные — в одном файле **`storage\database.sqlite`**. Бэкап = просто скопировать
этот файл (можно при остановленном сервере). Плюс сохраните **`storage\secret.key`**
(нужен для расшифровки настроек). Эти два файла храните отдельно от GitHub.

## 6. Частые вопросы

- **Репозиторий «весит» ~90 МБ** — это из-за вшитого портативного PHP (`tools\php`), чтобы
  на новом ПК ничего не ставить. Если PHP уже установлен в системе и вы не хотите хранить его
  в git — добавьте строку `/tools/php/` в `.gitignore`, выполните
  `git rm -r --cached tools/php && git commit -m "не хранить PHP в git"`. Тогда `start.bat`
  будет использовать системный PHP (нужны расширения pdo_sqlite, mbstring, zip).
- **Случайно закоммитили секрет?** Удалите файл из истории и смените ключ/токен. Проще всего —
  не коммитить: всё лишнее уже перечислено в `.gitignore`.
- **Word/PDF для виз** работают только там, где установлен Microsoft Word (или LibreOffice).
  Это не зависит от git — конвертация идёт на машине, где запущен сервер.
