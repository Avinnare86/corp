# Корпоративный портал «Учёт работы специалистов»

Внутренний веб-портал организации: учёт и оплата работы специалистов (квота/анкеты и визы),
электронный документооборот (СЭД), расчёт зарплаты, кадры, табели, отпуска и служебки о стимуле.
Один вход — у каждого сотрудника своя рабочая область по его ролям.

- **Стек:** PHP 8.1+ (самописный MVC, без Composer), БД — SQLite (по умолчанию для разработки) или PostgreSQL (прод).
- **Запуск:** локально под Windows — портативный PHP в `tools\php` (`start.bat`); на сервере — Docker или nginx/Apache + php-fpm.
- **Документы Word/Excel:** формирование ходатайств, описей, гарантийных писем, реестров; конвертация в PDF — через MS Word (Windows) или LibreOffice (Linux).

## Модули

| Модуль | Что делает |
|--------|-----------|
| **Квота / анкеты** | Загрузка и распределение анкет, проверка специалистами, контроль качества (выборка + штрафы с эскалацией), повторная проверка брака, недельный норматив и доплата по тарифам стран. |
| **Визы** | Загрузка ходатайств из Word, ИИ-подстановка адресов, распределение и проверка, экспорт DOCX/PDF, **формирование описей и гарантийных писем по странам**, визовые указания МИД, возврат строк на доработку после отказа и вычеты. |
| **СЭД** | Входящие/исходящие/внутренние документы, корреспонденты, маршруты согласования/подписания/ознакомления (ЭП), поручения и контроль сроков, номенклатура дел и архив, рабочий стол с корзинами задач, штрихкод/QR на РК, реестры, обращения граждан (59-ФЗ). |
| **Расчёт зарплаты** | Гарантированный оклад+надбавка + сделка (анкеты по тарифам, операции), недельный норматив с доплатой сверх, стимул по служебкам, штрафы (не ниже минимума), отсечка 25-го числа, рейтинг. |
| **Кадры / табели / отпуска** | Карточки сотрудников и переводы, оргструктура и должности, электронный и месячный табель, график отпусков с согласованием и запретными зонами. |
| **Стимул** | Служебки о стимуле с маршрутом ЭП: начальник отдела → заместитель → директор → бухгалтерия; бюджет ФОТ. |

## Быстрый старт

### Локально (Windows, портативный PHP — ничего ставить не нужно)
```
git clone https://github.com/Avinnare86/corp.git
cd corp
start.bat
```
Откроется `http://localhost:8000`. БД (SQLite) и демо-данные создаются при первом запуске.
Демо-входы: `admin / admin123`, менеджер виз `vizman / vizman123`, специалист `petrov / petrov123`, контролёр `control / control123`.

### Сервер (Linux, Docker — приложение + PostgreSQL + LibreOffice)
```
git clone https://github.com/Avinnare86/corp.git
cd corp && cp .env.example .env   # задайте DB_PASS
docker compose up -d --build
```
Подробно — в **[docs/DOCKER.md](docs/DOCKER.md)**. ⚠️ После первого входа смените пароль `admin`.

### Сервер (Linux, без Docker)
PHP 8.1+ (`pdo_pgsql, mbstring, zip, openssl, json, fileinfo`), nginx/Apache + php-fpm, корень `public/`,
LibreOffice для PDF. Запрос к администратору сервера/БД — в **[Письмо-администратору.md](Письмо-администратору.md)**,
полная процедура — в **[РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md)**.

## Структура проекта
```
app/Core         — ядро (роутер, БД, аутентификация, шаблоны)
app/Controllers  — обработчики запросов
app/Services     — бизнес-логика (расчёт ЗП, выборка, парсинг, документы)
app/Views        — шаблоны (partials/ — переиспользуемые блоки)
app/routes.php   — таблица маршрутов
database/        — миграция схемы (migrate.php) и стартовые данные (seed.php)
public/          — единая точка входа (index.php), статика
storage/         — БД SQLite, загрузки, ключ шифрования (вне public, не в git)
tools/php        — портативный PHP для Windows (в Docker/Linux не используется)
docs/            — руководство пользователя, инструкция по Docker
```

## Обновление
```
git pull
# Windows:  tools\php\php.exe database\migrate.php
# Linux/Docker: миграция применяется автоматически при старте (или php database/migrate.php)
```
Миграция идемпотентна — данные сохраняются.

## Безопасность
В репозиторий **не попадают** секреты и данные (см. `.gitignore`): `storage/secret.key` (ключ шифрования настроек),
рабочая БД, `*.bas`, образцы документов с персональными данными. Ключ OpenRouter и пароли SMTP хранятся
в зашифрованных настройках, пароль БД — в переменных окружения.
