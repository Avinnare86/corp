## 1. База данных (PostgreSQL)

Просьба создать **пустую** базу и пользователя и прислать параметры подключения.

- PostgreSQL **12+**, база в кодировке **UTF-8** (в данных кириллица).
- Пользователь — **владелец базы** (или `CREATE` в схеме `public` + полные права на таблицы): приложение само создаёт и обновляет таблицы при первом запуске и обновлениях.

```sql
CREATE ROLE uchet LOGIN PASSWORD '‹надёжный_пароль›';
CREATE DATABASE uchet
    OWNER uchet
    ENCODING 'UTF8'
    LC_COLLATE 'ru_RU.UTF-8'
    LC_CTYPE   'ru_RU.UTF-8'
    TEMPLATE   template0;
```

**Пришлите нам:** хост, порт (обычно 5432), имя базы, пользователя, пароль (защищённым каналом), нужен ли **SSL** к БД, метод аутентификации в `pg_hba.conf` (`scram-sha-256` или `md5`).

## 2. Размещение программы (Linux)

### 2.1. Откуда взять код — GitHub
Исходный код в репозитории: **https://github.com/Avinnare86/corp** (ветка `main`).
Репозиторий **приватный** — доступ для клонирования мы предоставим (на ваш выбор): добавим ваш GitHub-аккаунт как читателя **или** пришлём read-only токен. Сообщите, что удобнее, и (если по токену) — он подставляется вместо пароля при `git clone`.

```bash
# клонирование на сервер (пример каталога)
sudo git clone https://github.com/Avinnare86/corp.git /var/www/corp
# приватный репозиторий: логин — Avinnare86, пароль — предоставленный токен
```
> Примечание: в репозитории есть папка `tools/php` — это портативный PHP **для Windows**, на Linux она не нужна (используется системный PHP), можно не трогать.

### 2.2. PHP и веб-сервер
- **PHP 8.1+ (рекомендуется 8.3)** + расширения: `pdo_pgsql`, `mbstring`, `zip`, `openssl`, `json`, `fileinfo`, `xml`.
  Ubuntu/Debian:
  ```bash
  sudo apt install php8.3-cli php8.3-fpm php8.3-pgsql php8.3-mbstring php8.3-zip php8.3-xml
  ```
- Веб-сервер **nginx + php-fpm** (или Apache). **Корень сайта — папка `public/`**, единая точка входа `public/index.php`; все запросы перенаправлять на `index.php`. Пример для nginx:
  ```nginx
  root /var/www/corp/public;
  index index.php;
  location / { try_files $uri $uri/ /index.php?$query_string; }
  location ~ \.php$ { include snippets/fastcgi-php.conf; fastcgi_pass unix:/run/php/php8.3-fpm.sock; }
  ```
- **Права на запись:** папка `storage/` должна быть доступна на запись пользователю php-fpm (обычно `www-data`):
  ```bash
  sudo chown -R www-data:www-data /var/www/corp/storage
  ```
  Там создаётся ключ шифрования `secret.key` (его нельзя удалять/перезаписывать), загрузки и временные файлы.
- **Подключение к БД — через переменные окружения** (пароль только в окружении, не в файлах). Задать в пуле php-fpm (`/etc/php/8.3/fpm/pool.d/www.conf`):
  ```
  env[DB_DRIVER] = pgsql
  env[DB_HOST] = ‹host›
  env[DB_PORT] = 5432
  env[DB_NAME] = uchet
  env[DB_USER] = uchet
  env[DB_PASS] = ‹пароль›
  ```
- **Первый запуск — создать схему и стартовые данные:**
  ```bash
  cd /var/www/corp
  DB_DRIVER=pgsql DB_HOST=‹host› DB_PORT=5432 DB_NAME=uchet DB_USER=uchet DB_PASS='‹пароль›' php database/migrate.php
  # и (по желанию демо-данные):
  DB_DRIVER=pgsql ... php database/seed.php
  ```
- **Обновления:** `git pull` в каталоге проекта, затем `php database/migrate.php` (миграция идемпотентна, данные не теряются).

### 2.3. LibreOffice — для PDF виз, описей и гарантийных писем
Приложение формирует документы в Word (.docx) и конвертирует их в PDF через **LibreOffice в headless-режиме** (команда `soffice`). Поэтому на сервере приложения нужно:
- Установить LibreOffice (достаточно Writer):
  ```bash
  sudo apt install libreoffice-writer
  ```
- **Шрифты** (для кириллицы и Times New Roman — описи используют этот шрифт):
  ```bash
  sudo apt install fonts-liberation fonts-crosextra-carlito fonts-crosextra-caladea
  # при необходимости точного Times New Roman:
  sudo apt install ttf-mscorefonts-installer
  ```
  (Liberation Serif метрически совместим с Times New Roman — разметка сохранится.)
- Пользователь php-fpm (`www-data`) должен иметь право запускать `soffice` и писать во временную папку (приложение создаёт отдельный профиль LibreOffice на каждый запуск).
- Проверка от имени www-data:
  ```bash
  sudo -u www-data soffice --headless --convert-to pdf --outdir /tmp /var/www/corp/storage/templates/gp.docx
  ```
  Должен появиться PDF в `/tmp`.

> Часовой пояс сервера — местное время (для корректных дат и расчёта «по 25-е число»): `sudo timedatectl set-timezone ‹ваш_TZ›`.

## 3. Сетевые настройки

1. **Доступ пользователей** к веб-порту приложения; рекомендуем публиковать по **HTTPS** (TLS-сертификат на внутреннее доменное имя), порт 443.
2. **Приложение → PostgreSQL:** разрешить подключение с сервера приложения к серверу БД на порт **5432** (фаервол + запись в `pg_hba.conf` для IP приложения).
3. **На время развёртывания** — исходящий **HTTPS к github.com** с сервера приложения (для `git clone`/`git pull`). Если apt-репозитории закрыты — доступ к зеркалу пакетов для установки PHP/LibreOffice.
4. **SSL к БД** — если БД принимает только SSL, сообщите, мы включим режим в строке подключения.
5. *(Опционально)* **SMTP** для email-уведомлений (хост, порт, логин/пароль, шифрование).
6. *(Опционально)* **openrouter.ai** (исходящий HTTPS) — только если будете использовать ИИ-подстановку адресов в визах; без него всё работает в локальной сети.
