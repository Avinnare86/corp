## 1. База данных (PostgreSQL)

Просьба создать **пустую** базу и пользователя и прислать нам параметры подключения.

**Требования:**
- PostgreSQL версии **12 или новее**.
- База в кодировке **UTF-8** (обязательно — в данных кириллица).
- Пользователь должен быть **владельцем базы** (или иметь право `CREATE` в схеме `public` + полные права на таблицы) — приложение само создаёт и обновляет таблицы при первом запуске и при обновлениях.

**Пример команд (psql):**
```sql
CREATE ROLE uchet LOGIN PASSWORD '‹надёжный_пароль›';
CREATE DATABASE uchet
    OWNER uchet
    ENCODING 'UTF8'
    LC_COLLATE 'ru_RU.UTF-8'   -- на Windows-сервере PG, напр. 'Russian_Russia.1251'; главное — ENCODING UTF8
    LC_CTYPE   'ru_RU.UTF-8'
    TEMPLATE   template0;
```

**Пришлите нам в ответ:**
1. Хост (IP или имя сервера БД): `____`
2. Порт (обычно 5432): `____`
3. Имя базы: `____`
4. Имя пользователя: `____`
5. Пароль: `____` *(желательно отдельным защищённым каналом)*
6. Требуется ли **SSL**-подключение к БД (да/нет; если да — какой режим)?
7. Метод аутентификации в `pg_hba.conf` (`scram-sha-256` или `md5` — оба подходят).

## 2. Размещение программы

**Что мы передадим вам:**
- Архив приложения (~90 МБ): исходный код + при необходимости портативный PHP для Windows; **или** доступ к git-репозиторию (по вашему предпочтению).
- Файл `secret.key` и рабочую БД мы **не передаём** — ключ шифрования генерируется на сервере автоматически.

**Уточните, пожалуйста, среду размещения — Windows или Linux:**

**Вариант А. Windows-сервер**
- Дополнительно ставить PHP не нужно — он идёт в комплекте (папка `tools\php`).
- Запуск: как служба (приложение слушает порт, по умолчанию 8000) либо за веб-сервером (IIS/nginx).
- Для формирования **PDF виз / описей / гарантийных писем** на этом сервере должен быть установлен **Microsoft Word**.

**Вариант Б. Linux-сервер**
- Установить **PHP 8.1+ (рекомендуется 8.3)** с расширениями: `pdo`, `pdo_pgsql`, `mbstring`, `zip`, `openssl`, `json`, `fileinfo`.
- Веб-сервер **nginx или Apache + php-fpm**; корень сайта — папка `public/`, единая точка входа `public/index.php` (нужны правила перенаправления всех запросов на `index.php`).
- Для формирования **PDF виз / описей / гарантийных писем** установить **LibreOffice** (пакет `libreoffice-writer`).

**Общее для обоих вариантов:**
- Папка `storage/` должна быть **доступна на запись** приложению (загрузки, временные файлы, файл-ключ `secret.key`). `secret.key` нельзя удалять/перезаписывать после создания.
- Подключение к БД задаётся **переменными окружения** (пароль — только в окружении, не в файлах):
  `DB_DRIVER=pgsql`, `DB_HOST=…`, `DB_PORT=…`, `DB_NAME=…`, `DB_USER=…`, `DB_PASS=…`
- Первый запуск: выполнить миграцию схемы (одна команда — создаст таблицы), затем загрузку стартовых данных. Инструкции приложим к архиву.
- Часовой пояс сервера — **местное время** (для корректных дат и расчёта «по 25-е число»).

## 3. Сетевые настройки

1. **Доступ пользователей к приложению.** Открыть пользователям сети доступ к веб-порту приложения. Желательно публиковать по **HTTPS** (TLS-сертификат на внутреннее доменное имя), порт 443; либо HTTP на ‹порт›.
2. **Приложение → PostgreSQL.** Разрешить подключение с сервера, где работает приложение, к серверу БД на порт **5432** (правило фаервола + запись в `pg_hba.conf` для IP приложения).
3. **SSL к БД.** Если БД принимает только SSL-соединения — сообщите, мы включим соответствующий режим в строке подключения.
4. *(Опционально)* **Почта.** Если нужны email-уведомления — параметры SMTP-сервера (хост, порт, логин/пароль, шифрование).
5. *(Опционально)* **Интернет-доступ.** Если будет использоваться ИИ-подстановка адресов в визах — разрешить исходящие **HTTPS-запросы к openrouter.ai** с сервера приложения. Без этого приложение полностью работает в локальной сети.
