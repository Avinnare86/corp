# Руководство пользователя

Инструкции по работе с порталом — общие действия и пошаговые сценарии **по каждой роли**.
У сотрудника может быть несколько ролей: тогда ему доступны разделы всех его ролей.
Администратор видит все разделы.

## Общее для всех

**Вход.** Откройте адрес портала, введите логин и пароль → попадёте на **Главную**. Выход — ссылка «Выход» в углу.

**Главная (дашборд).** Приветствие; ссылка на **расчётный листок**; блок **«Рабочий день»** (у кого ведётся явка);
карточки «Мои проекты» (разделы, где у вас есть задачи) и «Мои задачи» (входящие документы, поручения и пр.);
сводки **Сообщения** и **Уведомления**.

**Рабочий день (явка).** На главной: **«▶ Приступить к работе»** в начале смены, **«⏹ Завершить работу»** в конце
(день засчитывается в табель). Случайно закрыли — **«↻ Возобновить работу»**.

**Расчётный листок** (`/payroll`, ссылка на главной). Показывает за месяц: явку, сделку, гарантированный минимум
(оклад+надбавка), штрафы, стимул и **итог к выплате**. См. модель оплаты ниже.

**Чат** (`/chat` и всплывающий виджет 💬 в углу). «✚ Новый чат» → выбрать собеседника; вложения 📎 (до 10 МБ), эмодзи 😊.

**Уведомления** (`/notifications`). Приходят автоматически (согласования, входящие, решения). «прочитано» — отметить.

**Моя ЭП** (`/certs`). Управление своими электронными подписями (загрузка/удаление).

**Отпуска** (`/vacations`). Любой сотрудник подаёт заявку (даты «С / По», тип «новый период» или «изменение сроков»).
Маршрут: сотрудник → руководитель отдела (согласует, может поправить даты) → менеджер/админ (утверждает).
Пересечение с «запретной зоной» → автоотказ.

### Модель оплаты (кратко)
- **Гарантия:** оклад + надбавка, пропорционально отработанным дням (минимум к выплате).
- **Сделка:** проверенные анкеты по тарифам стран + операции (визы и пр.).
- **Недельный норматив (если задан в карточке):** оклад покрывает норматив анкат; **доплата по тарифу — только за анкеты сверх норматива**. Норматив 0 = всё по тарифу (чистый сдельщик); не задан = классическая модель `max(оклад, сделка)`.
- **Штрафы** (контролёр) уменьшают выплату, но **не ниже гарантии**; зафиксированные после 25-го переносятся на след. месяц.
- **Стимул** по утверждённым служебкам добавляется сверху.

## Специалист: проверка анкет
*(роль `anketa_worker`; раздел меню «Квота»)*

**Проверка досье** (меню Квота → «Проверка досье»):
1. Откройте рабочий день (кнопка на главной).
2. В списке непроверенных анкет по каждой: **«✓ Без замечаний»** (уходит в проверенные) или **«✎ Причины»** → отметить причины доработки → **«OK»**.
3. Массово: введите размер пачки, **«+ пачка»** (отметит первые N) или галочки вручную → **«✓ Без замечаний — выбранным»** / **«✎ Причины — выбранным»**.
4. Вкладка **«Проверенные»** — история; **«✎ Изменить»** причины, **«↩ В работу»** — вернуть в очередь.

**Мой норматив** (меню Квота → «Мой норматив»): выберите месяц; видите недельный норматив, сколько проверено, покрыто окладом и сверх норматива (за что идёт доплата), выполнение по неделям.

**Расчётный листок** — ваша зарплата за месяц.

## Контролёр
*(роль `controller`; раздел меню «Квота» → «Контроль анкет»)*

1. **«Сформировать выборку»** → выберите день (по умолчанию вчера). Система случайно берёт заданный % проверенных анкет по каждому специалисту; **все повторные проверки (брак) включаются обязательно**.
2. **«Открыть»** выборку. По каждой анкете:
   - **«Корректно»** → «Сохранить» (строка зелёная);
   - **«Ошибка»** → выберите тип ошибки (со штрафом) → «Сохранить» (строка красная). Повтор того же типа в месяце дороже (эскалация).
3. При ошибке система автоматически ставит анкету на **повторную проверку другим** специалистом (исходного исключает).
4. Когда все размечены — **«Завершить проверку и уведомить сотрудников»**.

## Менеджер проекта: анкеты
*(роль `anketa_manager`; раздел меню «Квота»)*

**Распределение** (Квота → «Распределение»):
1. Загрузка списка: название → **«📎 Выбрать файл»** (DOCX/XLSX/CSV/TXT, парсятся рег. номера КОД-НОМЕР/ГОД) → **«Загрузить»** (дубликаты пропускаются).
2. Перенос анкет между «🗑 Не распределено» и специалистами: фильтры (список/страна/поиск) → галочки или «+ пачка» → **«Передать →»**. Повторную проверку нельзя отдать тому, кто допустил брак.
3. Быстрое распределение — одному или поровну между несколькими; «Снять распределение по сотруднику» — вернуть в пул.

**Отчёт** (Квота → «Отчёт»): всего/проверено/не распределено, по сотрудникам и спискам, **«📊 Выгрузить в Excel»**.

**Норматив** (Квота → «Норматив»): по каждому специалисту задайте недельный норматив (поле + «OK»): пусто = классическая модель, число = доплата только сверх норматива, 0 = чистый сдельщик. Видны выработка по неделям, покрыто/сверх/доплата.

**Справочники** (Квота): **Страны** (коды и тарифные группы), **Причины** доработок (чек-листы для специалистов), **Ошибки** (типы и суммы штрафов).

## Специалист: проверка виз
*(роль `visa_worker`; раздел меню «Визы»)*

**Проверка виз** (Визы → «Проверка виз»):
1. Откройте рабочий день. Видите грид назначенных строк (по 10) с полями анкеты, в т.ч. адрес.
2. Заполните/выверьте поля → **«✓ Сохранить и показать следующие»** — строки помечаются проверенными (засчитываются в сделку «Виза — этап 2»).
3. Если строка вернулась на доработку — она снова появится в гриде с пометкой (в т.ч. «Отказ МИД»).

**Отработанное** (Визы → «Отработанное») — история проверенных по дням и партиям. **Рейтинг** — ваше качество.

## Менеджер проекта: визы
*(роль `visa_manager`; раздел меню «Визы»)*

**Загрузка и распределение** (Визы → «Загрузка и распределение»):
1. Загрузите .docx-ходатайства (название партии + файлы). Дубликаты по исх. № не грузятся.
2. **«▶ ИИ-адреса»** — пакетная подстановка адресов (если настроен OpenRouter).
3. Распределите строки между проверяющими (доска переноса: пул ↔ специалист, «+ пачка», «Передать →»).
4. Экспорт партии: **«⬇ DOCX (ZIP)»**, **«⬇ PDF (как в Word)»**. «⚙ Бланк» — даты/подписант партии. «📋 Строки» — реестр с выгрузкой выбранных.

**Формирование описей** (Визы → «Формирование описей») — из **проверенных** анкат, кросс-партийно по странам:
1. Выберите **страну** → отметьте кандидатов (проверенные, ещё не в описи).
2. Укажите **ФИО и должность подписанта** (по умолчанию из настроек) → **«Сформировать опись»**.
3. На карточке описи — **«⬇ Документы»** (ОПИСЬ + ОПИСЬ-СП + ГП + список Excel). До внесения указания можно убрать строку или удалить опись (вернуть анкеты в кандидаты).

**Описи / Указания** (Визы → «Описи / Указания»): реестр описей. На карточке после ответа МИД внесите **№ и дату визового указания**; отметьте **отказанные** строки → они уходят «на доработку», по каждой выберите вычет с первичного проверяющего (стоимость проверки или 0). Остальные → «указание получено».

**МИД: на доработке** (Визы → «МИД: на доработке»): распределите вернувшиеся строки специалистам (первичному проверяющему — нельзя). Повторная проверка оплачивается как новая.

**Отчёт** и **Рейтинг** — прогресс и качество. **Каталог операций** — стоимость операций («Виза — этап 2» и др.).

## Специалист: операции
*(роль `piecework_worker`; меню «Визы/операции (учёт)» → `/piecework`)*

Ввод сдельных операций (визовые этапы и доп. работы): выберите операцию, дату и количество → сохранить;
строку можно удалить. Всё попадает в сделку расчётного листка. **Расчётный листок** — ваша зарплата.

## Делопроизводство / СЭД
*(роль `docs_manager` — полное управление; участвуют все сотрудники как подписанты/исполнители; раздел меню «Документы»)*

**Документы** (`/docs`): регистрация входящих/исходящих/внутренних, выбор корреспондента, вложения.
Маршрут по документу: **согласование → подпись → ознакомление** (электронной подписью); каждый участник видит свой шаг
и подписывает/согласует/знакомится. На карточке — печать РК со **штрихкодом/QR**, журналы и **реестры**.

**Поручения** (`/orders`): создать поручение/резолюцию (в т.ч. по документу), назначить исполнителя и срок;
исполнитель отмечает выполнение; постановка **на контроль** со сроком и напоминаниями.

**Номенклатура дел** и **Архив** (`docs_manager`): реестр дел по подразделениям со сроками хранения; списание документов в дело и в архив.

**Обращения граждан** (59-ФЗ) и **Замещение** (передача задач при отсутствии) — у `docs_manager`.

**Рабочий стол** — корзины задач: на подписи / на согласовании / на ознакомлении / поручения на исполнении / на контроле (дублируется на главной в «Мои задачи»).

## Руководители
*(роли `dept_head`, `deputy_director`, `director`)*

**Служебки о стимуле** (меню «Стимул» → `/memos`): создание служебки на сотрудников отдела (виды стимула, суммы/проценты).
Маршрут ЭП: **начальник отдела → заместитель директора → директор → бухгалтерия**. На своём шаге — подписать/направить дальше или отклонить.
Утверждённый стимул попадает в расчётный листок сотрудника.

**Отпуска**: согласование заявок сотрудников отдела (с правом правки дат), для утверждающих — утверждение графика.

**Документы и поручения**: создание, подписание/согласование (ЭП), контроль исполнения.

## Бухгалтерия
*(роль `accountant`; раздел меню «Кадры» + «Стимул»)*

- **Сотрудники** → карточка → **установка надбавки** (только бухгалтерия).
- **Покрытие табелями** — контроль, что все табели закрыты, перед расчётом.
- Просмотр расчётных листков и **служебок о стимуле** (бухгалтерия — финальный узел маршрута стимула, перед выплатой).
- Может быть утверждающим по отпускам.

---

## Кадры и табель
*(роли `hr`, `hr_manager`, `timekeeper`; раздел меню «Кадры»)*

- **Сотрудники** (`hr`, `hr_manager`): карточки, приём/правка, **перевод** (должность/отдел — с логом и перерасчётом).
- **Оргструктура** и **Должности** (`hr_manager`): подразделения, иерархия, штатные должности и оклады.
- **Эл. табель** (`timekeeper`, `hr_manager`): ведение явки по дням; **Табель (месяц)** (`hr_manager`): месячный табель (норма/отработано). **Покрытие табелями** — контроль заполнения.
- **Отпуска**: согласование/утверждение, управление **запретными зонами**, формирование уведомлений по утверждённым.

---

## Финансы
*(роль `finance_manager`; раздел меню «Финансы»)*

- **Бюджет ФОТ** — лимиты и источники зарплатного фонда (контроль при назначении стимула).
- **Основания стимула** — каталог видов стимула. **Тарифы** — стоимость анкет по тарифным группам стран. **Доплаты** — каталог фиксированных надбавок.

## Администратор
*(роль `admin`; «Админ-панель»)*

Доступны все разделы. Дополнительно:
- **Сотрудники** — создание учёток, **назначение ролей** (через роли сотрудник получает доступ к разделам), сброс пароля, активность.
- **Оргструктура / Должности / Типы** — справочники кадров.
- **Настройки расчёта** (`/admin/settings`): процент выборки контроля, эскалация штрафа, дневная норма, **ФИО и должность подписанта описей/ГП по умолчанию**, ключ и модель **OpenRouter** (ИИ для виз), **SMTP** (email-уведомления).
- **Страны / Группы цен (тарифы)**, **Каталог операций**, **Причины / Ошибки** — справочники проектов.
- **Журнал действий** (`/audit`) — все мутации пользователей.

> Первое включение роли: откройте карточку сотрудника и назначьте нужные роли — соответствующие разделы появятся в его меню при следующем входе.

---

*Точные названия пунктов меню и кнопок соответствуют интерфейсу; если чего-то не видите — у вашей учётной записи нет соответствующей роли (обратитесь к администратору).*
